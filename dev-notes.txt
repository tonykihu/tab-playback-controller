TAB PLAYBACK CONTROLLER — Development Notes
=============================================
Date: 2026-02-26
Version: 1.4.0

PROJECT OVERVIEW
================
Chrome extension (Manifest V3) to control audio/video playback across
multiple browser tabs. Targeting Chromium-based browsers.
Goal: Free core + paid Premium/Pro tiers.


WHAT WAS BUILT (v1.0 → v1.4)
==============================

v1.0 — Starting Point
- Basic play/pause per tab
- Pause All / Play All buttons
- Persistent state via chrome.storage.local
- Simple white popup UI

v1.1 — Bug Fixes + Background Worker + Shortcuts
- Fixed: script.js had invalid JS (raw HTML attribute pasted as code)
- Fixed: Favicon fallback path pointed to nonexistent icons/icon16.png → icon.png
- Fixed: index.html had mail: instead of mailto: in contact link
- Fixed: <script> tag was outside </html>
- Added: Background service worker (event-driven, no polling)
  - Only wakes on chrome.tabs.onUpdated (audible/status changes) and onRemoved
  - Zero CPU when idle
- Added: Keyboard shortcuts
  - Alt+Shift+M — toggle current tab
  - Alt+Shift+P — pause all
  - Alt+Shift+O — play all
  - Customizable at chrome://extensions/shortcuts

v1.2 — Focus on Tab + Domain Grouping
- Click tab title in popup → switches browser to that tab
- Tabs grouped by domain when from multiple sites (YouTube, Spotify, etc.)
- Collapsible group headers with playing count
- Friendly labels for 15+ known media domains
- Smarter media detection: requires valid src + duration > 0
  (fixes false positives like ChatGPT hidden audio elements)

v1.3 — Volume Sliders + Licensing System
- Per-tab volume sliders (0-100%)
- Licensing module (license.js):
  - 3 tiers: Free, Premium, Pro
  - Feature gating via License.hasFeature()
  - Stored in chrome.storage.sync (persists across devices)
  - Activation modal in popup
  - Test keys: TPC-PREM-* for Premium, TPC-PRO-* for Pro
- Volume sliders visible but locked for free users (conversion tactic)
- Clicking locked slider opens upgrade modal
- Tier badge in header (gold for Premium, purple for Pro)

v1.4 — Dark/Light Theme Toggle
- Sun/moon toggle button in popup header
- CSS custom properties for all colors
- Theme preference persisted in chrome.storage.local
- Both themes work across all UI elements (tabs, sliders, modals, groups)

ARCHITECTURE DECISIONS
======================
1. Service worker is event-driven only — no setInterval, no alarms
   Wakes on tab events, writes to storage, sleeps again.

2. Popup reads from chrome.storage.local, doesn't duplicate listeners
   Uses chrome.storage.onChanged to stay in sync with background.

3. Media detection validates src + duration to avoid false positives
   (ChatGPT, etc. have hidden audio elements with no content)

4. License stored in chrome.storage.sync (not local) so it syncs
   across devices automatically.

5. CSS uses custom properties for theming — adding a new theme
   only requires adding a new set of variable overrides.
